Chunk Generation

- generating neighbours -


given a pre-generated chunk (e.g. on the left)

[2, 4, 5, 7, 8]
[4, 4, 5, 6, 7]
[5, 5, 5, 5, 6]
[7, 6, 5, 5, 5]
[8, 7, 6, 5, 5]



step 1.Generating corners
Corners are also generated randomly (λ), expect on the ones adjacent to a pre-existing chunk, i.e.

[2, 4, 5, 7, 8] -> [8, _, _, _, λ]
[4, 4, 5, 6, 7] -> [_, _, _, _, _]
[5, 5, 5, 5, 6] -> [_, _, _, _, _]
[7, 6, 5, 5, 5] -> [_, _, _, _, _]
[8, 7, 6, 5, 5] -> [5, _, _, _, λ]



step 2.Filling intermediate tiles
intermediate tiles are generated following the same algorithm

note: since the two "west" corners are the same as the adjacent chunk's, the first column is predictable

[2, 4, 5, 7, 8] -> [8, _, _, _, λ]
[4, 4, 5, 6, 7] -> [7, _, _, _, _]
[5, 5, 5, 5, 6] -> [6, _, _, _, _]
[7, 6, 5, 5, 5] -> [5, _, _, _, _]
[8, 7, 6, 5, 5] -> [5, _, _, _, λ]

The same process is used when generating new chunks to the "south"
The top two coords (N-W & N-E) are copied from the chunk "above" (S-W & S-E respectively)
