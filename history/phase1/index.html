Chunk Generation

- generation algorithm -


chunk starts as an empty array (e.g. with size=5)
[_, _, _, _, _]



step 1.Generating random corners
[1, _, _, _, 4]



step 2.Filling intermediate tiles
Each tile progresses steadily toward the opposite corner
In the example above, the "chunk" is 5 tiles wide, meaning each tile is 1/5 superior to its left neighbour

-> [1, +0.25Δ, +0.50Δ, +0.75Δ, 4]
This value is then multiplied by the difference between the two corners (4-1 = 3):
-> [1, 1.75, 2.5, 3.25, 4]
Finally values are rounded to allow easier formatting
-> [1, ~2, ~3, ~3, 4]



step 3.Rendering the heightmap
The resulting array (called a "Heightmap") is formatted into color-coded tiles
The higher the integer, the higher the "altitude"
Low numbers would represente the ground (or water for example) and higher numbers would be mountains (or plains above water)
